# 🔥 BUFF SKILL - Quick Start Guide

## What is BUFF?

BUFF is an autonomous code quality enhancement protocol that transforms your project through three stages:
1. **Debug** - Find and fix all errors
2. **Enhance** - Add improvements and features
3. **Validate** - Ensure everything works

## How to Use It

### Basic Commands

```
buff this                    → Full enhancement protocol
buff security                → Security focus only
buff performance             → Performance optimization only
buff testing                 → Testing improvements only
buff [file.py]              → Enhance specific file + dependencies
buff [directory]            → Enhance directory + dependencies
buff security and testing   → Multiple focus areas
buff but skip docs          → Exclude specific areas
```

### What It Does

When you say "buff [something]", the system:
- ✅ Finds and fixes all syntax/semantic errors
- ✅ Validates Windows path handling
- ✅ Improves test coverage
- ✅ Enhances based on your focus area
- ✅ Validates everything works
- ✅ Reports all improvements

## Available Focus Areas

| Focus | Description |
|-------|-------------|
| `security` | Path validation, input sanitization, attack prevention |
| `performance` | Algorithm optimization, caching, batch processing |
| `testing` | Unit tests, integration tests, coverage improvement |
| `reliability` | Error handling, graceful degradation, recovery |
| `features` | New export formats, filtering, visualization |
| `organization` | Code structure, design patterns, helpers |
| `docs` | Documentation, docstrings, guides |
| `monitoring` | Logging, metrics, performance tracking |
